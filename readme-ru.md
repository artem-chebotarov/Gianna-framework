# Gianna Framework

> ⚠️ **Внимание!**
>
> Gianna Framework находится в стадии активной разработки. Внутреннее устройство фреймворка может меняться очень часто. Если после обновления перестал работать код, который работал вчера, вероятно, были внесены изменения в ядро проекта, и потребуется адаптация приложения под новую версию.

---

# Что такое Gianna Framework

**Gianna Framework** — это JavaScript-фреймворк для быстрой разработки логики фронтенд-приложений.

Основная задача фреймворка — выступать в роли оболочки приложения и управлять его жизненным циклом. Вопросы дизайна и внешнего оформления остаются на стороне разработчика.

---

# Зачем был создан проект

Проект начинался как эксперимент и способ повысить собственные навыки программирования. Со временем он вырос в полноценный фреймворк и стал доступен на GitHub для всех желающих.

---

# Возможности

- Шаблоны на основе XML-файлов (`*.gtpl`)
- Встроенная поддержка мультиязычности
- Маршрутизация страниц (routing)
- Система очистки ресурсов **Gianna Garbage Collector**
- Работа с IndexedDB из коробки
- Собственная система обработки кликов и пользовательских действий
- Поддержка плагинов
- Управление жизненным циклом приложения
- Специальные конструкции внутри шаблонов (условия, циклы и другие управляющие элементы)
- Безопасное взаимодействие шаблонов с сервером через ядро фреймворка

---

# Планируемые возможности

- **Моментальность** — точечное обновление интерфейса без тяжёлых реактивных вычислений
- Поддержка JSON с комментариями

```json
{
  /* комментарий */
}
```

# Философия проекта

Мне надоели тяжёлые и перегруженные решения.

Я хочу иметь возможность быстро исправить шаблон даже на слабом устройстве: часах, терминале или кассовом киоске.

Gianna Framework не создаётся для конкуренции с существующими решениями. Его цель — упростить жизнь разработчикам и сделать веб-приложения менее ресурсоёмкими.

# Быстрый старт

Предполагается, что каталог `gianna-framework` расположен на вашем сервере.

## index.html

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <title>Document</title>
</head>
<body>
    <div id="index"></div>
    <script src="./main.js" type="module"></script>
</body>
</html>
```

Контейнер `#index` используется для загрузки приложения.

## index.gtpl

```xml
<template>
    <config>
    </config>

    <meta>
    </meta>

    <content>
        <header>
            <p>MyApp</p>
        </header>

        <main>
            <h1>Hello world</h1>
        </main>

        <footer>
            <p>Copyright</p>
        </footer>
    </content>
</template>
```

### Структура шаблона

| Тег | Назначение |
|------|------|
| `<template>` | Корневой элемент шаблона. Обязателен. |
| `<config>` | Конфигурация шаблона. |
| `<meta>` | Метаинформация. |
| `<content>` | Основное содержимое. |

## main.js

```javascript
import { $gn, $gianna } from "./gianna-framework/gf.js";

(async () => {
    try {
        await $gn.loadConfig({
            lang: "en"
        });

        await $gn.view.render(
            "#index",
            "index.gtpl",
            true
        );
    } catch (e) {
        console.error(e);
    }
})();
```

### Параметры

#### `$gn.loadConfig()`

Загружает конфигурацию приложения.

#### `$gn.view.render()`

```javascript
$gn.view.render(selector, template, clearContainer);
```

| Параметр | Описание |
|-----------|-----------|
| `selector` | CSS-селектор контейнера |
| `template` | Путь к `.gtpl` шаблону |
| `clearContainer` | Очистить контейнер перед рендерингом |
|
